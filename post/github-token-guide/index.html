#日计-DIY #GitHub Token #登录教程

第一次使用
先把 GitHub Token 准备好

如果你是第一次打开 `日计-DIY`，这篇文章会带你从 GitHub 设置页一路走到网页登录成功

当前这版 `日计-DIY` 最稳的登录方式是 `Personal access token (classic)` + `repo` 权限 先按这篇文章走通，后面再考虑更细的权限模型会轻松很多
现在打开 GitHub Token 页面 回到日计-DIY

前言

`日计-DIY` 当前通过 `GitHub Token` 同步账本。也就是说，网页本身是公开入口，但账本数据还是放在你自己的 GitHub 仓库里

所以第一次使用时，你需要先在 GitHub 里创建一条能访问仓库的 token，再把它粘贴回登录页

开始前你需要准备什么

  • 一个 GitHub 账号
  • 能正常登录 GitHub 网页
  • 已经打开 `日计-DIY` 登录页

操作步骤

G-01

进入 Token 页面

你可以直接打开 GitHub 的 token 设置页，也可以按菜单路径慢慢点进去

头像 Settings Developer settings Personal access tokens Tokens (classic)

直接链接： https://github.com/settings/tokens

G-02

新建 classic token

  1. 点击 `Generate new token`
  2. 选择 `Generate new token (classic)`

如果 GitHub 要你重新输入密码或做二次验证，按提示完成就可以

G-03

填写基本信息

`Note` 建议写成后面一眼能认出的名字，比如 `日计-DIY 网页登录`

`Expiration` 建议第一次先选 `30 days`、`90 days` 或 `1 year`

G-04

勾选权限

当前这版最稳的做法是只先勾：

repo

因为网页登录后需要创建账本仓库、读取账本内容和写入同步内容，而这些都依赖 `repo`

G-05

生成并马上复制

  1. 滚动到页面最下方
  2. 点击 `Generate token`
  3. 看到 token 后立刻复制并保存
这一刻最重要
GitHub 只会完整显示这一次，关掉页面后就不能再把原文找回来
G-06

回到日计-DIY 登录

  1. 回到 `日计-DIY` 登录页
  2. 点击 `使用 GitHub Token 开始`
  3. 粘贴刚才复制的 token

正常情况下，登录弹层会关闭，然后你就可以继续创建或同步账本

G-07

如果刚生成就忘了复制

不用找回原文，直接回到 token 列表删掉旧 token，再重新生成一条新的最快

G-08

如果网页登录失败

  1. 检查是不是贴错了 token
  2. 检查是不是生成成了 fine-grained token
  3. 检查是不是漏勾了 `repo`
  4. 检查 token 有没有过期
  5. 检查复制时有没有多带空格或换行

常见问题

为什么不推荐一开始就用 fine-grained token

因为当前这版 `日计-DIY` 是公开网页 + GitHub Token 的组合，classic token 在首次跑通时更稳，排障成本也更低

只勾 `repo` 真的够吗

对当前网页登录场景，一般够用。如果后面真的遇到权限报错，再按具体报错补判断更稳

如果我卡住了怎么办

直接把这几个信息发出来就行：
文档：`GitHub Token 教程`
步骤号：例如 `G-04`
现象：例如 “登录后提示权限不足”
