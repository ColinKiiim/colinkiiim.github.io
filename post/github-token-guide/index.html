#日计-DIY #GitHub #教程

第一次使用
先把 GitHub Token 准备好

如果你第一次打开 `日计-DIY`，大概率会先被 `GitHub Token` 这一步卡住。这篇文章就只做一件事：把这一步带你走通

前言

`日计-DIY` 现在通过 `GitHub Token` 同步账本。也就是说，网页本身是公开的，但你的账本数据还是放在你自己的 GitHub 仓库里

这也是它和一般记账 App 不太一样的地方。你不是注册一个平台账号，而是先在 GitHub 里生成一条 token，再把它贴回登录页

听起来有点陌生，但实际做一次并不复杂。当前这版最稳的做法是用 Personal access token (classic)，并勾选 repo 权限

开始前你需要准备什么

  • 一个 GitHub 账号
  • 能正常登录 GitHub 网页
  • 已经打开 `日计-DIY` 登录页

如果这些都没问题，下面就可以直接开始了

操作步骤

G-01

进入 Token 页面

最直接的方式就是打开这个页面： https://github.com/settings/tokens

如果你更习惯从菜单走，也可以按这个路径进入：头像 → Settings → Developer settings → Personal access tokens → Tokens (classic)

G-02

新建 classic token

  1. 点击 `Generate new token`
  2. 选择 `Generate new token (classic)`

如果 GitHub 要你重新输入密码或做二次验证，按提示完成就可以，这一步很正常

G-03

填写基本信息

Note 建议写成你后面一眼能认出来的名字，比如 日计-DIY 网页登录

Expiration 第一次先选 30 days、90 days 或 1 year 都可以

G-04

勾选权限

当前这版最稳的做法，是只先勾一个权限： repo

原因也不复杂。登录之后，网页需要帮你创建账本仓库、读取里面的数据、再把同步结果写回去，而这些动作都依赖 repo

G-05

生成并马上复制

  1. 滚动到页面最下方
  2. 点击 `Generate token`
  3. 看到 token 后立刻复制并保存

这里一定要注意，GitHub 只会完整显示这一次。你一旦把这个页面关掉，后面就不能再回来看原文了

G-06

回到日计-DIY 登录

  1. 回到 `日计-DIY` 登录页
  2. 点击 `使用 GitHub Token 开始`
  3. 粘贴刚才复制的 token

正常情况下，登录弹层会关闭，然后你就可以继续创建或同步账本

G-07

如果刚生成就忘了复制

不用浪费时间去找回原文。最省事的做法就是回到 token 列表，把刚才那条删掉，再重新生成一条新的

G-08

如果网页登录失败

  1. 检查是不是贴错了 token
  2. 检查是不是生成成了 fine-grained token
  3. 检查是不是漏勾了 `repo`
  4. 检查 token 有没有过期
  5. 检查复制时有没有多带空格或换行

如果你还是卡住，后面直接把步骤号发给我就行，比如 `我卡在 G-04`

常见问题

为什么不推荐一开始就用 fine-grained token

因为当前这版 `日计-DIY` 是公开网页 + GitHub Token 的组合，classic token 在第一次跑通时更稳，排障成本也更低

只勾 `repo` 真的够吗

对当前网页登录场景，一般够用。如果后面真的遇到权限报错，再按具体报错补判断更稳

如果我卡住了怎么办

直接把这几个信息发出来就行：文档名 `GitHub Token 教程`、步骤号、你看到的现象，比如 `登录后提示权限不足`
