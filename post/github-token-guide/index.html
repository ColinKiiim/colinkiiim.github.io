第一次使用 先把 GitHub Token 准备好

这篇文章只讲一件事：生成一条能在日计-DIY 登录里使用的 GitHub classic token

先说结论

直接创建 Personal access token (classic)

权限先只勾 repo

开始前

  • 准备一个已经能正常登录的 GitHub 账号
  • 同时打开日计-DIY 登录页，生成后可以直接粘贴回去
  • 如果你以前生成过 fine-grained token，这次不要选它

操作步骤

G-01 进入 Token 页面

直接打开 https://github.com/settings/tokens

如果从 GitHub 菜单进入，路径是：头像 → Settings → Developer settings → Personal access tokens → Tokens (classic)

G-02 新建 classic token

  1. 点击 Generate new token
  2. 选择 Generate new token (classic)

如果 GitHub 要你重新输入密码或做二次验证，按提示完成

G-03 填写基本信息

Note 可以写成你后面一眼能认出来的名字，比如 日计-DIY 网页登录

Expiration 第一次选 30 days、90 days 或 1 year 都可以，过期后这条 token 就不能继续用了，需要重新生成一条新的再更换

G-04 勾选权限

当前这版最稳的做法，是先只勾一个权限：repo

登录之后创建账本、读取账本和写入同步内容，都要用到这个权限

G-05 生成并复制

  1. 滚动到页面最下方
  2. 点击 Generate token
  3. 看到 token 后立刻复制并先保存好

GitHub 只会完整显示这一次

G-06 回到日计-DIY 登录

  1. 回到日计-DIY 登录页
  2. 点击 使用 GitHub Token 开始
  3. 粘贴刚才复制的 token

正常情况下，登录弹层会关闭，然后你就可以继续创建或同步账本

G-07 如果忘了复制

直接回到 token 列表，把刚才那条删掉，再重新生成一条新的最快

登录失败时

  1. 确认你生成的是 classic token，不是 fine-grained token
  2. 确认权限里勾了 repo
  3. 确认复制时没有多带空格或换行
  4. 确认 token 还没过期
  5. 如果还是不确定，直接删除旧 token 再重新生成一条
