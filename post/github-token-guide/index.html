第一次使用 先把 GitHub Token 准备好

这篇文章只解决一个问题：第一次使用 `日计-DIY` 时，怎么把 GitHub Token 准备好

开始前你需要知道什么

当前这版 `日计-DIY` 通过 `GitHub Token` 同步账本，所以你第一次使用时，需要先到 GitHub 里生成一条 token，再把它贴回登录页

最稳的做法是使用 Personal access token (classic)，并勾选 repo 权限

你需要准备什么

  • 一个 GitHub 账号
  • 已经能正常登录 GitHub 网页
  • 已经打开 `日计-DIY` 登录页

操作步骤

G-01 进入 Token 页面

直接打开： https://github.com/settings/tokens

如果你想从 GitHub 菜单进去，路径是：头像 → Settings → Developer settings → Personal access tokens → Tokens (classic)

G-02 新建 classic token

  1. 点击 `Generate new token`
  2. 选择 `Generate new token (classic)`

如果 GitHub 要你重新输入密码或做二次验证，按提示完成就可以

G-03 填写基本信息

Note 建议写成你后面一眼能认出来的名字，比如 日计-DIY 网页登录

Expiration 第一次先选 30 days、90 days 或 1 year 都可以

G-04 勾选权限

当前这版最稳的做法，是只先勾一个权限：repo

因为网页登录之后，需要创建账本仓库、读取账本内容和写入同步内容，而这些动作都依赖 repo

G-05 生成并复制

  1. 滚动到页面最下方
  2. 点击 `Generate token`
  3. 看到 token 后立刻复制并保存

GitHub 只会完整显示这一次，关掉页面后就不能再把原文找回来

G-06 回到日计-DIY 登录

  1. 回到 `日计-DIY` 登录页
  2. 点击 `使用 GitHub Token 开始`
  3. 粘贴刚才复制的 token

正常情况下，登录弹层会关闭，然后你就可以继续创建或同步账本

G-07 如果忘了复制

直接回到 token 列表，把刚才那条删掉，再重新生成一条新的最快

G-08 如果登录失败

  1. 检查是不是贴错了 token
  2. 检查是不是生成成了 fine-grained token
  3. 检查是不是漏勾了 `repo`
  4. 检查 token 有没有过期
  5. 检查复制时有没有多带空格或换行

常见问题

为什么不推荐一开始就用 fine-grained token

因为当前这版 `日计-DIY` 是公开网页 + GitHub Token 的组合，classic token 在第一次跑通时更稳

只勾 `repo` 真的够吗

对当前网页登录场景，一般够用。如果后面真的遇到权限报错，再按具体报错补判断更稳

如果我卡住了怎么办

直接把步骤号发给我就行，比如 `我卡在 G-04`
